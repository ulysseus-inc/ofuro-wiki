# パターン C：HTTP のみ（お試し・開発用）

## このパターンの特徴

| 項目 | 内容 |
|------|------|
| **条件** | 証明書・ドメイン不要 |
| **HTTPS** | ❌ HTTP のみ |
| **ブラウザ警告** | なし |
| **クリップボード** | ⚠️ 制限あり（下記参照） |
| **リバースプロキシ** | 不要 |
| **向き不向き** | 動作確認、デモ、開発環境、個人の localhost 利用 |

## クリップボードの制限について

ブラウザのセキュリティ仕様により、**HTTP 環境では `navigator.clipboard` API が使用できません**。
ofuro-wiki はこの制限に対してメモリ内フォールバックを実装していますが、以下の制限があります：

| 操作 | 動作 |
|------|------|
| **同一タブ内** でのコピー＆ペースト | ✅ 動作する |
| **別タブ・別ウィンドウ** からのペースト | ❌ 動作しない |
| **OS のクリップボード**（他アプリへのペースト）| ✅ プレーンテキストとして動作する |
| テキスト選択・右クリックコピー（ブラウザ標準） | ✅ 動作する（テキストのみ） |

> **本番利用には向きません。** 複数人での利用、またはコピー＆ペーストをフル活用したい場合は
> [パターン A](./pattern-A.md) または [パターン B](./pattern-B.md) を選択してください。

---

## セットアップ手順

まず [共通セットアップ](./README.md#共通セットアップ手順) を完了させてから、以下を行ってください。

`.env` の `BASE_URL` を設定します：
```bash
# localhost からのアクセスの場合
BASE_URL=http://localhost:3010

# 別のPCからアクセスする場合（IPアドレスを指定）
BASE_URL=http://<server-ip>:3010
```

---

## 起動

リバースプロキシは不要です。Docker Compose をそのまま起動すれば使えます：

```bash
docker compose up -d
```

`http://localhost:3010`（または `http://<server-ip>:3010`）にアクセスします。

---

## localhost アクセスについて

**自分自身のPC（localhost）** からアクセスする場合は、`navigator.clipboard` が使用可能です
（localhost は HTTPS と同等のセキュアコンテキストとして扱われるため）。

```bash
BASE_URL=http://localhost:3010
```

この場合、クリップボードはフル機能で動作します。

---

## ポート変更

デフォルトはポート `3010` です。変更する場合は `.env` に追記します：

```bash
# .env
APP_PORT=8080
```

`docker-compose.yml` の `ports` も合わせて変更が必要な場合があります。
