# セキュリティポリシー

## 脆弱性の報告

ofuro-wiki のセキュリティを真剣に受け止めています。脆弱性を発見した場合は、**公開の Issue や Pull Request では報告しないでください**（攻撃に悪用される恐れがあります）。

代わりに、以下のいずれかの非公開チャネルでご連絡ください。

- **メール**: contact-ml@ulysseus.co.jp（件名に「ofuro-wiki security」と明記してください）
- **GitHub Security Advisories**: 本リポジトリの「Security」→「Report a vulnerability」から非公開で報告

### 報告に含めていただきたい情報

- 脆弱性の種類と影響範囲
- 再現手順（PoC があれば歓迎）
- 対象のバージョン / コミットハッシュ
- 想定される攻撃シナリオ

## 対応の流れ

1. 受領後、できるだけ速やかに受付確認を返信します。
2. 影響範囲を調査し、修正方針を検討します。
3. 修正をリリースし、報告者の希望に応じてクレジットを記載します。

責任ある開示にご協力いただけますようお願いいたします。修正がリリースされるまでは、詳細を公開しないでください。

## 対象範囲

- 本リポジトリのコード（フロントエンド / バックエンド）
- 公式の Docker イメージおよびデプロイ設定

サードパーティ依存（AFFiNE / BlockSuite 等の上流、各 npm パッケージ）の脆弱性については、まず各上流プロジェクトへの報告をご検討ください。ofuro-wiki 側での対応が必要な場合は上記チャネルへご連絡ください。
